

## How to Use This Repository

You do **not** need to be a programmer to review this material.

Suggested review order:
1. Review the PDF of the presentation slides and refeer to the **prompts** to understand how tasks were described to AI  
2. Review the **HTML files** to see how a simple web app is structured (JavaScript within the HTML files) 
3. Examine the **JSON file** to understand data organization  
4. Look through the **JavaScript analysis script** to see how loops and functions operate on real data  

You are encouraged to:
- Ask an AI tool to explain any code line-by-line
- Modify variables or inputs and observe the results
- Treat this as a sandbox for learning

---

## Key Takeaways

- AI can meaningfully lower the barrier to coding for non-coders  
- Understanding **data structure** is more important than memorizing syntax  
- Simple scripts can unlock complex scholarly workflows  
- AI-generated code must be **understood, tested, and validated**  
- AI should augment—not replace—clinical and academic expertise  

---

## Limitations & Responsible Use

- Follow all institutional and privacy guidelines  
- Do not use AI-generated code blindly in clinical or high-risk settings  
- Avoid relying on AI for statistical inference without domain expertise  
- Always test with known inputs and expected outputs
